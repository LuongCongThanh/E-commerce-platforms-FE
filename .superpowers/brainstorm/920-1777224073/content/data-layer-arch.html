A — Hooks tách riêng

Data tĩnh trong data/. Hooks trong hooks/ lo filter/transform/sort. Components chỉ nhận props hoặc gọi hook — không bao giờ import data trực tiếp.

Pros

  • Swap sang API thật chỉ thay body trong hook
  • Mỗi layer test độc lập
  • Component hoàn toàn presentational

Cons

  • Nhiều file hơn một chút
_lib/
home.ts ← data + hooks cùng file
products.ts
categories.ts
_components/
SectionBestSellers.tsx ← import hook từ _lib/

B — Colocated (data + hook cùng file)

Mỗi domain (home, products) có 1 file trong _lib/ chứa cả static data lẫn hook. Ít file hơn, dễ tìm hơn khi project còn nhỏ.

Pros

  • Ít file hơn, đơn giản hơn
  • Dễ trace: 1 file = 1 domain

Cons

  • File phình to khi thêm logic
  • Khó tách test riêng data vs hook
home/page.tsx ← Server Component
const data = getHomeData() ← call ở server
<SectionBestSellers products={data.bestSellers} />

_lib/
getHomeData.ts ← pure function, no hooks
data/home.ts
_components/
SectionBestSellers.tsx ← Server Component, chỉ render

C — Server-first (không hooks)

Page server component gọi data functions, truyền props xuống các section. Không dùng hooks trừ khi cần interactivity (add-to-cart). Sạch nhất, ít JS nhất.

Pros

  • Zero JS bundle cho sections tĩnh
  • Dễ hiểu: data flow từ trên xuống
  • RSC-native pattern

Cons

  • Khi nối API thật cần refactor page
  • Sections không tự fetch được

💡 Recommendation: A — hooks tách riêng là pattern chuẩn nhất để migrate sang real API sau này mà không đụng components.
